## ユーザーが作成した設定

ユーザーが本ソフトウェアを使用して作成した設定ファイル（`config.yml`）には、profile 定義、パス glob、git identity、`gh` CLI アカウント参照などが含まれますが、その著作権はすべて作成者に帰属します。本ライセンスは gospelo-identity ソフトウェア自体にのみ適用され、ユーザーが作成した設定内容には適用されません。

## 外部ツールおよび認証情報

本ソフトウェアは、ユーザーに代わって外部ツール（`git`、`gh` CLI）を呼び出し、ローカルリポジトリの設定の読み書き、および GitHub アカウントコンテキストの切替を行います。gospelo-identity の開発者は、これらの外部ツールの動作および当該ツールが管理する認証情報の安全性について、いかなる表明も保証も行いません。以下はすべてユーザーの責任となります:

- `switch` 実行前に予定される変更を確認すること（`--dry-run` でプレビュー、または事前に `check` で対象 profile を確認）
- GitHub の認証情報（`gh auth login`）およびアクセストークンのスコープ・有効期間を、適用される法令・規制、GitHub の利用規約、およびユーザー自身のセキュリティポリシーに従って管理すること
- `switch` で選択された GitHub アカウントによって行われたコミット、プッシュ、リリース、その他の行為

## アカウント所有権の非検証

gospelo-identity は、ローカルに設定された git identity および `gh` CLI のアクティブアカウントが、現在の作業ディレクトリに対してユーザーが宣言した profile と一致しているかを検証するに過ぎません。GitHub アカウントの所有権、雇用主からの利用許可、または何らかの契約上の義務については一切検証しません。profile を選択することは、ユーザーが既に保有していない権利を新たに付与するものではありません。各設定アカウントで公開する権利を有していることの確認は、すべてユーザーの責任となります。
